What is SOTA
in Regulatory AI?

State-of-the-Art (SOTA)란 "현재 기술 수준에서 달성 가능한 최고의 성능과 방법론"을 의미합니다.

우리의 시스템은 단순히 최신 AI 모델을 쓰는 것을 넘어,
1. 법적 표준 준수(Compliance), 2. 문맥 무결성(Context), 3. 의미론적 정확성(Semantics)을
동시에 해결하는 Engineered SOTA Architecture입니다.

STANDARD PARSING

Document Drafting Handbook (DDH)

법률 문서는 아무렇게나 쓰여지지 않습니다. 미국 NARA의 Document Drafting Handbook이라는 엄격한 표준에 따라 작성됩니다.

우리의 Hierarchy Extractor는 이 핸드북의 규칙(Section 2-15)을 코드로 옮겨, Section > Paragraph(a) > Paragraph(1)의 구조를 정확하게 인식합니다. 이는 AI 모델보다 더 정확한 Rule-based SOTA 접근 방식입니다.

hierarchy_extractor.py 
class HierarchyExtractor:
    # DDH (Document Drafting Handbook) 표준 패턴 정의
    HIERARCHY_LEVELS = {
        1: {
            "name": "chapter", 
            "pattern": r"^(Chapter\s+[IVX]+)"
        },
        2: {
            "name": "section", 
            "pattern": r"^\d+\.\d+"  # 예: 101.1 (CFR 표준)
        },
        3: {
            "name": "paragraph", 
            "pattern": r"^\([a-z]\)"   # 예: (a) (DDH 규격)
        }
    }

Parent-Child RAG

검색의 딜레마(정밀도 vs 문맥)를 해결하는 State-of-the-Art 전략입니다.

우리의 코드는 문서를 쪼갤 때, 검색용 Child Chunk와 답변용 Parent Chunk를 동시에 생성하고 연결합니다. AI는 작은 조각을 찾아내지만, 사용자에게는 전체 조항(Section)을 보여주어 문맥 왜곡 없는 답변을 제공합니다.

semantic_chunker.py 
def create_chunks(self, section):
    # [Parent] 전체 문맥 보존 (LLM 답변용)
    parent = ProcessedChunk(
        chunk_id=f"parent_{section['id']}",
        context_text=section['full_text'],
        chunk_type="parent"
    )
    
    # [Child] DDH 구조 기반 세분화 (검색용)
    for para in section['paragraphs']:
        yield ProcessedChunk(
            text=para, 
            parent_id=parent.chunk_id,  # 부모 연결
            chunk_type="child"
        )
MODEL SOTA

BGE-M3 Hybrid Embedding

"전자담배"와 "ENDS"가 같은 의미임을 아는 것(Dense)과, "0.7mg"이라는 수치를 정확히 찾는 것(Sparse)은 다릅니다.

본 시스템은 이 두 가지를 동시에 수행하는 SOTA 모델(BGE-M3)을 탑재하여, Dense Vector와 Sparse Vector를 결합한 하이브리드 검색을 수행합니다.

embedding_pipeline.py 
class EmbeddingPipeline:
    def __init__(self):
        # Current SOTA Model: BGE-M3
        self.model = BGEM3FlagModel('BAAI/bge-m3')

    def generate_vectors(self, text):
        output = self.model.encode(
            text, 
            return_dense=True,  # 의미 검색 (Semantic)
            return_sparse=True  # 키워드 매칭 (Lexical)
        )
        return output

# 시스템 아키텍처

본 문서는 REMON(Regulation Monitoring & Mapping System)에서 **아키텍처**를 정리합니다.

* **에이전트 서비스 플로우**: AI 에이전트들이 순차적으로 동작하는 처리 흐름
* **계층적 서비스 아키텍처**: 배포 단위 기준의 서비스 역할 분리

---

## 1. 아키텍처 범위

본 문서에서 다루는 범위는 다음과 같습니다.

* 규제 문서가 시스템에 입력된 이후의 처리 흐름
* AI 에이전트 간 단계적 호출 순서
* Frontend / Backend / AI / Data 간 책임 분리

다음 항목은 본 문서에서 다루지 않습니다.

* 세부 알고리즘 및 프롬프트 구현
* 성능 최적화 및 운영 정책
* 추상적인 설계 원칙 또는 장기 로드맵

---

## 2. 계층적 서비스 아키텍처

![계층적 서비스 아키텍처](./images/service-architecture.png)

* **Frontend**
  사용자 입력, 처리 상태 표시, 결과 리포트 조회

* **Backend API**
  요청 제어, 상태 관리, AI 서비스 호출

* **AI Service**
  규제 분석 에이전트 실행 및 워크플로우 관리

* **Data Storage**
  규제 문서, 분석 결과, 메타데이터 저장

각 계층은 독립적인 서비스로 배포되며, 역할만을 기준으로 분리됩니다.

---

## 3. 에이전트 서비스 플로우

![에이전트 서비스 플로우](./images/agent-flow.png)

에이전트 서비스 플로우는 규제 문서 처리의 **실제 실행 순서**를 의미합니다.

일반적인 처리 흐름은 다음과 같습니다.

1. 규제 문서 입력
2. 전처리 및 구조화
3. 규제 변경 감지
4. 제품 매핑
5. 영향도 분석 및 리포트 생성

각 단계는 하나의 에이전트 또는 에이전트 그룹으로 구성되며,
이전 단계의 출력만을 입력으로 받아 동작합니다.

# REMON 문서 개요

이 디렉토리는 REMON 프로젝트의 **상세 설계 문서**를 관리합니다.

메인 README는 프로젝트의 개요와 방향성을 빠르게 이해하기 위한 용도이며,
아키텍처, AI 워크플로우, 데이터 모델, 판단 통제(HITL)와 같은 깊은 내용은 본 문서들에서 다룹니다.

---

## 문서 구성

### 1. 시스템 아키텍처

REMON 전체 시스템의 구조와 구성 요소 간 역할을 설명합니다.

* 규제 문서 수집 및 전처리 파이프라인
* AI 분석 및 워크플로우 오케스트레이션
* 데이터 저장 및 검색 구조
* Frontend – Backend – AI 서비스 간 연계 구조

→ `architecture.md`

---

### 2. AI 워크플로우

REMON에서 사용되는 AI 파이프라인과 에이전트 기반 처리 흐름을 설명합니다.

* 규제 변경 감지 로직
* 규제 문서 구조화 전략
* 제품 매핑 및 영향도 분석 흐름
* 리포트 생성 단계

→ `ai-workflow.md`

---

### 3. 데이터 모델

규제 문서와 제품 정보를 저장·관리하기 위한 데이터 설계를 설명합니다.

* PostgreSQL 스키마 설계
* 메타데이터 및 이력 관리 방식
* Vector DB(Qdrant) 활용 구조
* 엔티티 간 관계 정의

→ `data-model.md`

---

### 4. Human-in-the-Loop (HITL)

AI 판단 결과에 대한 사람 개입 설계와 그 이유를 설명합니다.

* HITL이 필요한 배경
* 사람이 개입하는 지점과 기준
* 승인 / 수정 / 보류 흐름
* 오탐·미탐 리스크 통제 관점

→ `hitl.md`

---

각 문서는 독립적으로 읽을 수 있도록 작성되며,
전체를 순서대로 읽으면 REMON의 설계 의도와 판단 기준을 단계적으로 이해할 수 있습니다.
